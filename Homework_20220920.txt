1. Зачем нужен GIT и какие задачи решают с его помощью?

Git — это система управления версиями. У Git две основных задачи: первая — хранить информацию о всех изменениях в вашем коде, начиная с самой первой строчки, а вторая — обеспечение удобства командной работы над кодом.

2. Что такое репозиторий GIT и где он находится?
 
Репозиторий Git можно создать,  выбрав любую папку на локальном компьютере, либо создать на github.

3. Для чего используется файл .gitignore?

. gitignore нужен для скрытия файлов и папок от системы контроля версий Git.

4. Что такое "коммит"?

«commit» - сохранение, фиксация  изменений в программном коде .

5. Укажите базовые команды в рамках работы с GIT:

Настройка аккаунта локального пользователя: 
git config --global user.name 'name'
git config --global user.email 'email'

Создание репозитория.
git init (создать новый реп)

Добавление файлов в индекс и выполнение коммита.
git add . (добавить в очередь на коммит все файлы)
git commit -m 'Любой комментарий к комиту'
git remote add origin <скопированная_ссылка>

Просмотр журнала коммитов.
git status
